= Creating your project and pipeline server
include::_attributes.adoc[]

準備段階として、皆さんはそれぞれ次の作業を行います

. データ サイエンス プロジェクトを作成します
** これにより、作業内容をまとめることができます

. データ接続を作成します
** パイプライン サーバーが成果物を格納するために必要です

. データ サイエンス パイプライン サーバーをデプロイします
** 1 つ必要になりますが、最初から作成しておくことをお勧めします

. ワークベンチを起動します
** コンテンツとノートブックを確認するために使用します

. Git リポジトリをワークベンチにクローンします
** これにはプロトタイプのすべてのコードが含まれます

以下の手順で、これらの手順を説明します。注意深く従ってください。

== プロジェクトの作成

* まず、OpenShift AI ダッシュボード アプリケーションで、左側の [**Data Science Projects**] メニューに移動します:
+
[.bordershadow]
image::02/02-02-ds-proj-nav.png[]

* ユーザー ID と同じ名前のプロジェクトを作成します
** 一意のユーザー ID が割り当てられています: `{user}`
** 次に、まったく同じ名前のプロジェクトを作成する必要があります: `{user}`
+
重要: 割り当てられたユーザーは {user} です。間違えると後で問題が発生します
* リソース名は変更しないでください
* 必要に応じて、プロジェクトの説明に姓と名を入力します
* 次のようになります:
+
[.bordershadow]
image::02/02-02-create-project.png[]
+
重要: スクリーンショットのように `userX` にしないでください (`X` は数字にする必要があります)

== パイプライン サーバーのデータ接続を作成します

* ここでは、目的に応じて、シンプルなオブジェクト ストレージとして機能するように、クラスターに Minio のインスタンスをデプロイしました。
* *Data connections* タブに移動して、 [**Add data connection**] する必要があります。

+
[.bordershadow]
image::02/02-02-add-dc.png[]

* 入力する必要がある情報は次のとおりです:
** Name:

[.console-input]
[source, text]
[subs=attributes+]
Shared Minio - pipelines
** Access Key:
[.lines_space]
[.console-input]
[source, text]
[subs=attributes+]
{minio-user}
** Secret Key:
[.lines_space]
[.console-input]
[source, text]
[subs=attributes+]
{minio-pass}
** Endpoint:
[.lines_space]
[.console-input]
[source, text]
[subs=attributes+]
{minio-endpoint}
** Region:
[.lines_space]
[.console-input]
[source, text]
[subs=attributes+]
none
** Bucket:
[.lines_space]
[.console-input]
[source, text]
[subs=attributes+]
{user}
+
重要: もう一度言いますが、使用するバケットは、提供されたユーザー ID と一致する必要があります

== パイプライン サーバーの作成

ワークベンチを作成する前に、パイプライン サーバーを作成することを強くお勧めします。それでは、今すぐ作成しましょう!　*Pipelines* タブに移動します。 `Create Pipeline`　ではなく　`Configure Pipeline`　が表示される場合があります。それを選択して、minio エンドポイント　`http://minio.ic-shared-minio.svc.cluster.local:9000/`　とバケット　`{user}`　を入力します。ユーザー名とパスワードも入力する必要がある場合は、それぞれ　`minio`　と　`minio123`　です。

* データ サイエンス プロジェクト (DSP) の *Pipelines* タブで、[**Create pipeline Server**] または[**Configure pipeline Server**] をクリックします。
+
[.bordershadow]
image::02/02-02-pipelineserver01.png[]

* 以前に作成したデータ接続 (**Shared Minio - pipelines**) を選択し、[**Configure pipeline server**] ボタンをクリックします。
+
[.bordershadow]
image::02/02-02-pipelineserver02.png[]

* パイプライン サーバーの準備ができたら、画面は次のようになります。
+
[.bordershadow]
image::02/02-02-pipelineserver03.png[]

この時点で、パイプライン サーバーの準備が整い、デプロイされています。

重要: 画面の準備が整うまで**待​​つ**必要があります。まだ回転している場合は、完了するまでお待ちください。パイプライン サーバーの準備が**整う前に**ワークベンチの作成を続行すると、ワークベンチはパイプラインを送信できなくなります。
