== ワークベンチを起動

* データ接続とパイプライン サーバーが完全に作成されたら
* ワークベンチを作成します
+
[.bordershadow]
image::02/02-03-create-wb.png[]
* 次の特性があることを確認します:
** Choose a name for it, like: `My Workbench`
** Image Selection: `CUSTOM - Insurance Claim Processing Lab Workbench`
** Container Size: `Standard`
** Accelerator: `None`
* 次のようになります:
+
[.bordershadow]
image::02/02-02-launch-workbench-01.png[]
* その他のワークベンチ設定 (ストレージなど) を変更する必要はありません
* ワークベンチが完全に起動するまで待ちます
* 起動したら、**[開く]** リンクをクリックして接続します
+
[.bordershadow]
image::02/02-03-open-link.png[]

* 以前と同じ認証情報で認証します
* 次の設定を受け入れるように求められます:
+
[.bordershadow]
image::02/02-02-accept.png[]

* 同意します
* 次の画面が表示されます:
+
[.bordershadow]
image::02/02-02-jupyter.png[]

== Git 共通リポジトリのクローンを作成します

Git リポジトリのコンテンツをクローンして、プロトタイピング演習の一環として作成されたすべての資料にアクセスできるようにします。

* Git UI の使用:
** Jupyter で Git UI を開きます:
+
[.bordershadow]
image::02/git-clone-1.png[]
+
** Git リポジトリの URL を入力します:
+
[.console-input]
[source,adoc]
[subs=attributes+]
----
{git-clone-repo-url}
----
+
[.bordershadow]
image::02/git-clone-2.png[]

この時点で、プロジェクトは、私たちがやりたい作業を行う準備が整いました。
